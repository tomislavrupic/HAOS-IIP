Stage 21A — Pairwise Overlap Memory Probe

(Relational Registration Test)

Role in the program

Stage 20 remains the effective-law / boundary layer.
Stage 21 opens a new architectural ingredient: relational registration.

The core hypothesis is not stronger feedback, but a new state variable:

when two packets overlap in a sufficiently strong and persistent way,
the system creates a bounded shared memory of encounter
that can influence later evolution.

This is not a consciousness claim.
It is not an observer claim.
It is only a dynamical registration hypothesis.

⸻

Structural motivation

Stages 14–19 established:
	•	weak local extensions do not bind
	•	relational ordering does not become dynamics
	•	synchronization does not create persistence
	•	substrate memory remains regime-inert
	•	mixed scalar-vector reinforcement also remains persistence-inert

The shared structural gap is:

interactions leave traces,
but not a shared state of relation.

Stage 21A tests the smallest possible remedy:

pairwise overlap memory.

⸻

Architectural change

Introduce a bounded pairwise memory variable:

R_{ij}(t)

for each packet pair `(i, j)`.

Interpretation:
	•	`R_{ij}` stores encounter history
	•	it increases when packet overlap is sustained
	•	it decays otherwise
	•	it can weakly bias later interaction

No kernel topology change.
No new field sector.
No global coupling.

⸻

Memory update rule

Suggested minimal form:

R_{ij}(t + dt)
=
(1 - dt/\tau_R) R_{ij}(t)
+
\alpha_R \cdot H\big(O_{ij}(t) - O_*\big)
\cdot O_{ij}(t)

where:
	•	`O_{ij}(t)` = normalized overlap measure between packets `i` and `j`
	•	`O_*` = fixed overlap threshold
	•	`\tau_R` = memory decay timescale
	•	`\alpha_R` = weak accumulation coefficient
	•	`R_{ij}` is clamped to a small bounded interval

This keeps memory:
	•	local in pair space
	•	reversible
	•	bounded
	•	falsifiable

⸻

Feedback channel

Pair memory may influence later evolution only weakly.

Minimal option:
	•	modify effective pair coupling or local attraction proxy by a term proportional to `R_{ij}`

Example diagnostic form:
	•	interaction term gets a bounded factor `(1 + \epsilon_R R_{ij})`

Constraint:
	•	must not create global artificial stabilization
	•	must remain weaker than the base dynamics

⸻

Representatives

Use the same canonical three:
	1.	clustered_composite_anchor
	2.	phase_ordered_symmetric_triad
	3.	counter_propagating_corridor

No new geometries.
No new packet families.

⸻

Pilot structure

Minimal 9-run pilot:
	•	null
	•	very weak relational memory
	•	weak relational memory

across the three representatives.

Refinement only on earned promotion.

⸻

Primary observables

Persistence-first metrics:
	•	composite lifetime
	•	binding persistence
	•	coarse basin persistence
	•	corridor dwell / locking

Relational-memory diagnostics:
	•	peak `R_{ij}`
	•	mean `R_{ij}` over active pairs
	•	memory retention time
	•	number of registered pair episodes

Do not promote on memory signal alone.
Persistence metrics still dominate.

⸻

Success criteria

Promote only if at least one representative shows:
	•	clear composite lifetime gain or binding persistence gain
	•	and a nontrivial `R_{ij}` history
	•	and no global artificial stabilization
	•	and bounded memory dynamics

Label positive only as:
	•	candidate relational registration effect

⸻

Negative freeze rule

Freeze Stage 21A negative if:
	•	all runs show bounded but persistence-inert `R_{ij}`
	•	no lifetime or binding gain appears
	•	no selective stabilization appears
	•	refinement does not promote any case

Interpretation:
	•	interaction traces alone do not become relational state in this architecture

⸻

Boundary

Even a positive Stage 21A result means only:
	•	shared encounter history can influence later persistence

It does not imply:
	•	consciousness
	•	observation theory
	•	awareness
	•	spacetime

This is strictly a dynamical registration test.
